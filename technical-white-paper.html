单一件工作的计分方法

假设这件工作由 \(N\) 个作者 完成. 用上面的 UI, 每个成员自行提出他(和其他作者)在这件工作里所占的百分比 (%): \( p_1, p_2, ... p_n \)

对所有作者 提交的百分比 做加权平均。 To be valid, # of voters have to reach quorum.

The total tokens worth \( T \) for this task is also voted and weighted-averaged by authors.

Non-authors may also vote on this task: either by altering the percentages \( p_i \) or by altering total value \( T \), but it seems dubious that outsiders could influence the percentages, so let's not allow this option.

Exceptions could be raised to seek external advice / resolution.

资料库 (Git) 界面

Git 这边需要的是 将每个成员的贡献「可视化」出来，以方便其他人 判断他的贡献价值。

注意 我们是用 Git 的 API 读取 repository 资料，这并不依赖 GitHub 提供资料。

我们打算支缓这几种文件的可视化，因为它们的格式是很标准的：

  1. Text 档案 可视化
  2. SVG 可视化
  3. 网页 可视化

其他格式的文件，由于没有可视化，那就不能观察到每个人的局部贡献，只能当成一个整体 unit.

聊天室 (Discord) 界面

Discord 的目的是 管理「聊天室」，每个 cluster 有独立的聊天室，互相之间可以传递讯息：

  1. 将 Discord 的聊天内容搬到 微信 / QQ群组
  2. Allow users to create new clusters
  3. Allow users to send messages from cluster to cluster
